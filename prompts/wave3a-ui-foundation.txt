# Goal
Deliver the UI foundation that every subsequent Pulse LWC surface (workflow builder, record stepper, action hub, conversation hub) will depend on:

1. **Poppins font static resource** — zipped woff2 files + @font-face CSS that ship inside the managed package so subscribers don't need external CSP trust.

# Detailed scope

## 1. Poppins font static resource

Create `packages/pulse-core/force-app/main/default/staticresources/pulse_poppins_fonts/` (directory-style static resource, deploys as a zip).

Download Poppins weights 300, 400, 500, 600, 700, 800 (Latin subset only — no Latin Extended, Cyrillic, etc.; we ship English-only for now) from Google Fonts CDN using `curl`:

```
curl -sL -o pulse_poppins_fonts/poppins-<weight>.woff2 \
  "<CDN URL for that weight — fetch from https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;500;600;700;800&display=swap>"
```

The Google Fonts CSS at that URL enumerates the actual woff2 CDN URLs. Parse them and download each locally.

Create `pulse_poppins_fonts/fonts.css` with `@font-face` declarations that reference the local woff2 files relative to the static resource root:

```
@font-face {
  font-family: 'Poppins';
  font-style: normal;
  font-weight: 300;
  font-display: swap;
  src: url('./poppins-300.woff2') format('woff2');
}
/* … repeat for 400, 500, 600, 700, 800 */
```

Create `pulse_poppins_fonts.resource-meta.xml` alongside with `contentType = application/zip` and cacheControl Public.

## 2. Update pulse_brand_tokens.css

At the top of `packages/pulse-core/force-app/main/default/staticresources/pulse_brand_tokens.css`, add an `@import url('../pulse_poppins_fonts/fonts.css');` OR better, have consumers load both resources explicitly via loadStyle (see §4). Prefer the explicit-loader path because `@import` inside a static resource is fragile across Salesforce CDN caching.

## 3. Shared brand-token loader

Create `packages/pulse-core/force-app/main/default/lwc/pulseBrandTokens/pulseBrandTokens.js`:

```javascript
import { loadStyle } from 'lightning/platformResourceLoader';
import FONTS from '@salesforce/resourceUrl/pulse_poppins_fonts';
import TOKENS from '@salesforce/resourceUrl/pulse_brand_tokens';

let loadPromise;

export function loadPulseBrandTokens(lwc) {
    if (!loadPromise) {
        loadPromise = Promise.all([
            loadStyle(lwc, FONTS + '/fonts.css'),
            loadStyle(lwc, TOKENS + '/pulse_brand_tokens.css'),
        ]);
    }
    return loadPromise;
}
```

Create `pulseBrandTokens.js-meta.xml` as an isExposed=false LWC module (helper only, not a rendered component).

Every Pulse LWC will import this and call it in `connectedCallback()`:

```javascript
import { LightningElement } from 'lwc';
import { loadPulseBrandTokens } from 'c/pulseBrandTokens';

export default class PulseFoo extends LightningElement {
    connectedCallback() {
        loadPulseBrandTokens(this);
    }
}
```

## 4. LWC primitives

Build under `packages/pulse-core/force-app/main/default/lwc/` as six separate LWCs. Each must:
- Use the `.pulse-surface` wrapper class on the host element (via `.THIS :host` pattern in LWC CSS or manual classList setup in connectedCallback)
- Reference only CSS custom properties from pulse_brand_tokens.css — never hardcoded hex values
- Have API 66.0 in the meta xml
- Be exposed only internally (isExposed=false) — these are building blocks, not admin-addable components
- Have a `__tests__/` directory with a Jest-style test covering render + prop handling + accessibility (aria attributes, role, focus handling for interactive components)

### pulseButton
Props: `label` (String, required), `variant` (String: 'primary' | 'secondary' | 'dark' | 'ghost', default 'primary'), `size` (String: 'sm' | 'md' | 'lg', default 'md'), `disabled` (Boolean), `iconName` (String, optional — SLDS icon name for a leading icon). Emits `click` event. Renders the brand kit's `.pulse-btn` variant classes.

### pulseInput
Props: `label`, `value`, `placeholder`, `type` (text/email/number/password), `required`, `error` (String, optional — shows error text below input and applies error border), `helpText` (String, optional). Two-way binding via `value` prop and `change` event. Must render the label above, input with focus ring, optional error/help text below.

### pulseBadge
Props: `label`, `variant` ('purple' | 'magenta' | 'success' | 'warning' | 'error' | 'gray', default 'gray'). Static, no events. Renders the brand kit's `.pulse-badge` variant classes.

### pulseCard
Slotted content. Props: `heading` (optional), `iconHtml` (optional inline SVG or emoji for the card icon), `elevated` (Boolean, adds hover effect). Renders `.pulse-card` wrapper with optional `.pulse-card-icon` header.

### pulseToggle
The "one-checkbox" control from v4.1 §3.2. Props: `label` (String, required), `helpText` (String, optional), `checked` (Boolean), `disabled` (Boolean). Emits `change(detail: {checked: Boolean})`. Visually a switch (not a standard checkbox) — 44x24 pill with 20x20 thumb that slides between positions. Must meet WCAG AA contrast and support keyboard toggle (Space/Enter).

### pulseModal
Slotted content. Props: `open` (Boolean), `size` ('sm'|'md'|'lg'|'xl', default 'md'), `title` (optional heading), `dismissible` (Boolean, adds close X). Emits `close` event. Renders overlay + centered card with backdrop blur. Traps focus inside the modal while open. Escape key closes if dismissible.

## 5. Admin Studio shell

`packages/pulse-core/force-app/main/default/lwc/pulseAdminStudio/`:

Single LWC, exposed to Lightning App Builder and Tab. Renders a full-viewport layout:

- **Top bar**: Pulse wordmark (gradient-text treatment on the word "Pulse" — logo PNG assets not yet in repo), environment badge showing org id, user menu (placeholder).
- **Left sidebar**: nav items for "Workflow Builder", "Integrations Hub", "AI Config", "Library", "Settings". Icons from SLDS utility set. Active item uses the gradient background, inactive items use hover states.
- **Main panel**: swaps content based on active nav item. For this wave, each panel is a placeholder card with a heading ("Workflow Builder — coming soon") and a short description. The real panels arrive in Wave 3b/c.

State: local reactive `@track activeNav`. No Lightning Data Service, no Apex calls in this wave.

Must have a `pulseAdminStudio.js-meta.xml` with:
- `isExposed = true`
- `targets = lightning__AppPage, lightning__HomePage, lightning__Tab`
- `masterLabel = 'Pulse Admin Studio'`
