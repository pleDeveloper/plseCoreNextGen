### Cross-package visibility
`PulseConversationHubController` lives in the `pulse-conversations-core` package but calls `ProjectionWriteService` (in `pulse-core`). That's a cross-package call. `ProjectionWriteService.project(...)` is currently declared `public`. Since both packages share the `plse` namespace but are separate 2GP packages, `public` is insufficient — promote `ProjectionWriteService` and its DTOs (`ProjectionRequest`, `ProjectionResult`) to `global` so this controller compiles. Update the class and any callers that still compile. No logic change; visibility only.

Run `./scripts/check-global-discipline.sh` before committing to confirm no regression.

## 2. pulseConversationHub LWC

Path: `packages/pulse-conversations-core/force-app/main/default/lwc/pulseConversationHub/`

`pulseConversationHub.js-meta.xml`:
- `isExposed = true`
- `targets = lightning__RecordPage, lightning__AppPage, lightning__HomePage`
- `masterLabel = 'Pulse Conversation Hub'`

Props:
- `@api recordId` — auto-provided on record pages
- `@api instanceId` — for app-page usage where a workflow instance is in scope
- `@api targetObjectApiName = 'Account'` — override via app builder; used by acceptExtract

JS contract:
- `connectedCallback()` → `loadPulseBrandTokens(this)`.
- `@wire(recordId ? getConversationsForRecord : getConversationsForInstance, { recordId, workflowInstanceId })` — pick the right wire based on which prop is set.
- Reactive state: expandedConversationId (UI collapse/expand), expandedExtractId, selectedFieldsPerExtract (Map<extractId, Set<fieldKey>>).

Render (brand-compliant, no SLDS base):
- **Empty state**: pulseCard with "No conversations yet" message + subtle instructional copy.
- **Conversation list**: one pulseCard per Conversation, collapsed by default. Header shows:
  - medium as pulseBadge (Email/Call/Meeting/Chat → different variant: purple/magenta/gray)
  - subject as h4 with pulse-gradient-text on "Pulse" brandmark NOT applied here (that's marketing)
  - occurredAt as small muted date
  - Extract status summary: "N pending extracts", as pulseBadge warning if N>0
- **Expanded conversation**:
  - Turn transcript: one sub-card per TurnView. Role as badge (Customer/Agent/System/AI). Speaker + timestamp line. Content as pre-wrap text. Max-height with scroll if >400px total.
  - Extracts section: one expandable panel per extract.
    - "Facts proposed by AI" heading.
    - Per-fact row: fieldKey, value, confidence (rendered as a mini confidence bar — colored green/amber/red based on value thresholds 0.8/0.5/0.0 using semantic tokens).
    - Checkbox per fact (pulseToggle sized down) for select-to-accept.
    - Two pulseButtons at the bottom of the extract: "Accept selected" (variant primary, disabled if no selections) and "Reject" (variant ghost). "Accept selected" invokes acceptExtract with the checked field keys.
  - Footer actions: "Request re-extraction" (variant secondary) if status is Extracted/Rejected.
- **Post-action toasts**: use `alert()` fallback as with the other hubs (we're avoiding lightning-toast).

CSS: brand tokens only. Spacing tokens throughout. Confidence bar uses:
- ≥0.8 → `--pulse-success`
- ≥0.5 → `--pulse-warning`
- <0.5 → `--pulse-error`

# Discipline
- Brand tokens: every new LWC calls `loadPulseBrandTokens(this)`. No hardcoded hex outside `var()` fallbacks. No `lightning-*` base components.
- `global` surface: only what the Admin Studio shell (in another package) and any future UI needs. Every method of PulseConversationHubController must be `@AuraEnabled` and on a `global` class with `global` return types for LWC wire compatibility.
- Sentence case UI copy. Anti-buzzword. "Accept selected" not "Empower facts."
- No new CMDT, no new platform events, no new object fields in this wave. If a Status picklist value like "Partial" doesn't yet exist on `Conversation_Extract__c.Status__c`, add it as a value to the existing field metadata — that's an acceptable minor schema touch. Document the addition in the commit message.
