# Goal
Three deliverables, all in `packages/pulse-core/`:

1. `Stage_Dwell__c` fact object — append-only per-state-exit record for BI queries and predictor training.
2. `StageDwellHook` Apex TransitionHook implementation + registration mechanism — captures dwell times automatically as instances transition.
3. `StageDwellPredictor` Apex service — queries recent Stage_Dwell__c rows, computes median + p90 per (workflow_key, state_key, subject_kind), exposes `predictExitFor(instanceId)` returning an estimated exit datetime + confidence.
4. `pulseSlaHeatmap` LWC — surface the aggregated per-state dwell stats (median, p90, count) as a heatmap in the Admin Studio's "Reports" / "SLA" panel.

## 2. StageDwellHook (Apex)

Path: `packages/pulse-core/force-app/main/default/classes/StageDwellHook.cls`

```
public with sharing class StageDwellHook implements WorkflowRuntime.TransitionHook {

    // Registration helper — call from a post-install or setup script to
    // register the hook once per app lifetime. Runtime's static hooks list
    // persists for the transaction only, so external callers should re-register
    // per transaction. (This is a known limitation; see notes below.)
    public static void register() {
        WorkflowRuntime.registerHook(new StageDwellHook());
    }

    public void afterTransitions(List<WorkflowRuntime.TransitionContext> contexts) {
        // For each TransitionContext:
        //   - Look up (or create in-memory and buffer) a Stage_Dwell__c for the
        //     FROM state of this instance whose Exited_At is null → set
        //     Exited_At = stepResult.Completed_Date (or now()), compute
        //     Duration_Seconds, set Next_State_Key, Signal, Actor_Type.
        //   - Create a NEW Stage_Dwell__c for the TO state:
        //     Entered_At = stepResult.Completed_Date, Exited_At = null,
        //     State_Key = ctx.toStateKey, etc. (Terminal states get an entry
        //     row too; the predictor can filter if it wants.)
        //
        // Bulk pattern: aggregate all upserts, single DML at the end.
        //
        // If a FROM-state Stage_Dwell__c row does NOT exist for this instance
        // (first transition after legacy data, or tests that don't seed), skip
        // updating the FROM row and just create the TO row.
    }

    // ── Private helpers ──────────────────────────────────────────────
    // Build Stage_Dwell__c rows, look up open rows per instance, compute
    // duration in seconds.
}
```

### Registration during runtime
The `TransitionHook` list in `WorkflowRuntime` is a static Apex collection — it survives only for the current transaction. To make `StageDwellHook` always run, there are two options:

- **Option A (recommended for this wave):** modify `WorkflowRuntime.advance()` to auto-register the hook at the top of each call: `if (!hasStageDwellRegistered) { StageDwellHook.register(); hasStageDwellRegistered = true; }`. The `hasStageDwellRegistered` flag is a static Boolean inside WorkflowRuntime. One-line touch to `advance()`.

- **Option B:** expect every caller (LWC controllers, trigger code, queueables) to `StageDwellHook.register()` before calling `advance()`. More ceremony; error-prone.

Go with **Option A**. It's a 4-line delta in WorkflowRuntime that adds no cross-package dependency (StageDwellHook is in Core). Add a one-line comment explaining the auto-registration.

### StageDwellHookTest
Cover:
- Single transition creates TO row, marks FROM row exited, sets Duration_Seconds correctly.
- Batch of 5 instances transitioning simultaneously produces correct per-instance rows.
- Terminal transition sets Exited_At on FROM row AND creates a terminal TO row with Exited_At = null (the predictor can decide how to interpret).
- First-ever transition for a brand-new instance (no prior Stage_Dwell row for FROM state) only creates the TO row, no errors.

## 3. StageDwellPredictor (Apex)

Path: `packages/pulse-core/force-app/main/default/classes/StageDwellPredictor.cls`

```
global with sharing class StageDwellPredictor {

    global class StageStats {
        @AuraEnabled global String  workflowKey;
        @AuraEnabled global String  subjectKind;
        @AuraEnabled global String  stateKey;
        @AuraEnabled global Integer sampleCount;      // N rows used in this aggregation
        @AuraEnabled global Integer medianSeconds;
        @AuraEnabled global Integer p90Seconds;
        @AuraEnabled global Integer minSeconds;
        @AuraEnabled global Integer maxSeconds;
    }

    global class ExitPrediction {
        @AuraEnabled global Id       instanceId;
        @AuraEnabled global String   currentStateKey;
        @AuraEnabled global Datetime enteredAt;
        @AuraEnabled global Integer  medianSecondsForState;
        @AuraEnabled global Integer  p90SecondsForState;
        @AuraEnabled global Datetime expectedExitDate;   // enteredAt + medianSecondsForState
        @AuraEnabled global Datetime p90ExitDate;        // enteredAt + p90SecondsForState
        @AuraEnabled global Boolean  isOverdue;          // now() > p90ExitDate
        @AuraEnabled global Integer  sampleCount;
    }

    @AuraEnabled(cacheable=true)
    global static List<StageStats> aggregateDwellStats(
        String workflowKey,        // nullable — null = all workflows
        Integer lookbackDays       // default 90 if null/zero
    ) {
        // SOQL: SELECT Workflow_Key__c, Subject_Kind__c, State_Key__c, Duration_Seconds__c
        // FROM Stage_Dwell__c WHERE Exited_At__c != null AND Entered_At__c >= :cutoff
        // GROUP BY Workflow_Key__c, Subject_Kind__c, State_Key__c
        //
        // Aggregate stats computed in Apex since SOQL doesn't have median/p90.
        // Use the percentile algorithm: sort the N durations, pick index
        // floor(N * 0.5) for median, floor(N * 0.9) for p90.
        // Bounded to the 50,000-row SOQL cap — if lookback is large, warn
        // via a comment and accept truncation.
    }

    @AuraEnabled(cacheable=true)
    global static ExitPrediction predictExitFor(Id workflowInstanceId) {
        // Load the instance's current state + entered_at (from the open
        // Stage_Dwell__c row for this instance with Exited_At=null).
        // Look up stats for (workflow_key, subject_kind, state_key) via
        // aggregateDwellStats. Compute expectedExitDate = enteredAt + median,
        // p90ExitDate = enteredAt + p90. Return an ExitPrediction.
        //
        // If no dwell history exists for this state → return prediction with
        // sampleCount=0, nulls for the date fields (LWC can render "—").
    }
}
```

### StageDwellPredictorTest
- Seed 20+ Stage_Dwell__c rows across 2 states with known durations; assert median and p90 match.
- aggregateDwellStats filters by workflowKey + lookbackDays correctly.
- predictExitFor returns the right date arithmetic for an instance mid-flight.
- predictExitFor returns sampleCount=0 cleanly when no history exists for that state.
