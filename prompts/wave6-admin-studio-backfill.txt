# Goal — scope discipline
**Three panels + placeholder cleanup, read-only.** CMDT CRUD from Apex requires async Metadata API plumbing we already built for schema provisioning, but hooking each of these CMDTs into the Tooling API pipeline is too much for one wave. The panels are **read views** with deep links into Setup for editing. This matches v4.1's philosophy: make admins immediately see the current config state, defer the full in-app editor until those flows are load-bearing.

Each read-only panel has:
- Title + short description
- List of relevant records with key fields
- Badge indicators for `Active__c` / `Priority` / policy
- A muted "Edit in Setup →" link that opens `/lightning/setup/CustomMetadata/home` scoped to the right CMDT (open in a new tab)
- Empty state when no rows exist

## 2. LWCs — three new components

All paths under `packages/pulse-core/force-app/main/default/lwc/`.

### `pulseAiConfig/`

`js-meta.xml`:
- `isExposed = true`
- `targets = lightning__AppPage`
- `masterLabel = 'Pulse AI Config'`

JS / HTML behavior:
- `connectedCallback` loads brand tokens, wires `getAiConfig`.
- Renders three sections each as a pulseCard:
  - **AI Providers** — one sub-card per provider: name + model + Named Credential + priority badge + active/inactive status badge (success/gray).
  - **Tool Registrations** — one sub-card per tool: tool key + apex class + HITL policy as colored badge (Approval_Required = warning, Review_After = info, Autonomous = success, other = gray).
  - **Extraction Profiles** — list rows with profile key + medium badge + small muted "schema size: N chars" caption.
- Each section header has a subtle "Edit in Setup →" link that opens `/lightning/setup/CustomMetadata/home` in a new tab. Use a plain `<a target="_blank" rel="noopener">`; no lightning-* base components.
- Empty states for each section with useful instructional copy (not generic "no records").

### `pulseIntegrationsHub/`

`js-meta.xml` same targets/exposure as above.

Sections:
- **Named Credentials** — list with label, developerName, endpoint (truncated to 40 chars), principal type. Link to `/lightning/setup/NamedCredential/home`.
- **Channel Adapters** — list of Channel_Adapter_Registration__mdt rows.
- **Conversation Sources** — list of Conversation_Source_Registration__mdt rows.
- Same "Edit in Setup →" pattern for each.

### `pulseSettings/`

`js-meta.xml` same targets/exposure.

Sections:
- **Feature Flags** — each flag with a colored toggle-status pulseBadge (purple if enabled, gray if disabled) + description.
- **Pulse Permission Sets** — list with label + an "Assigned to you" pulseBadge when true, "Not assigned" muted text when false. Link to `/lightning/setup/PermSets/home`.
- **Recent Deployments** — last 10 from Deployment_Request__c, with target object + status pulseBadge (success/warning/error based on Completed/Pending/Failed) + completed date + error message preview when present.

### Tests
Each LWC gets a Jest test covering: empty state, populated state (at least one row per sub-section), section heading rendered, Setup link has `target="_blank"`.
