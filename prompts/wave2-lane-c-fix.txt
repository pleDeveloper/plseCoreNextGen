# Problem statement
`sf project deploy start -o pulse-core-next-dev --dry-run` currently returns Failed with ~350 component errors cascading from three files in the AI lane. ## Root cause 1: `PulseTool.cls` and `PulseAiProvider.cls` declare inner classes inside a Salesforce `interface`
Salesforce Apex does not allow inner class declarations inside an `interface` (unlike Java). Both files need to be restructured.

- `force-app/main/default/classes/PulseTool.cls` — currently declares `public interface PulseTool` with inner `class Context` and `class Result`. This cannot compile.
- `force-app/main/default/classes/PulseAiProvider.cls` — currently declares `public interface PulseAiProvider` with inner `class Request`, `class Response`, `class Message`, `class ToolCall`, `class ToolDefinition`, `class Usage`. This cannot compile.

### Required pattern — companion-class split
Keep each interface as a pure-method contract and move the DTOs to a companion class:

- Create `force-app/main/default/classes/PulseToolModels.cls` (+ `.cls-meta.xml`, API 66.0) with `public class PulseToolModels { public class Context { ... } public class Result { ... } }` — preserving every field, constructor (`Context(Id instanceId, Map<String, Object> input)`), and static factory (`Result.ok(String)` / `Result.fail(String)`).
- Rewrite `PulseTool.cls` to be:
  ```
  public interface PulseTool {
      String getToolKey();
      String getDescription();
      Map<String, Object> getInputSchema();
      String getDefaultHitlPolicy();
      PulseToolModels.Result execute(PulseToolModels.Context ctx);
  }
  ```
- Create `force-app/main/default/classes/PulseAiModels.cls` (+ `.cls-meta.xml`) with `public class PulseAiModels { public class Request { ... } public class Response { ... } public class Message { ... } public class ToolCall { ... } public class ToolDefinition { ... } public class Usage { ... } }` — preserving every field, constructor, and default (Request.maxTokens = 4096, Response.isError = false, etc.).
- Rewrite `PulseAiProvider.cls` to be:
  ```
  public interface PulseAiProvider {
      PulseAiModels.Response chat(PulseAiModels.Request req);
  }
  ```

### Propagate the rename through every caller
Update every reference to `PulseTool.Context` → `PulseToolModels.Context`, `PulseTool.Result` → `PulseToolModels.Result`, `PulseAiProvider.Request` → `PulseAiModels.Request`, `PulseAiProvider.Response` → `PulseAiModels.Response`, `PulseAiProvider.Message` → `PulseAiModels.Message`, `PulseAiProvider.ToolCall` → `PulseAiModels.ToolCall`, `PulseAiProvider.ToolDefinition` → `PulseAiModels.ToolDefinition`, `PulseAiProvider.Usage` → `PulseAiModels.Usage`.

## Root cause 2: `PulseRecordUpdateTool.cls` uses a reserved word
The file contains a local variable named `desc` (used as `Schema.DescribeSObjectResult desc = ...`). `desc` collides with SOQL's `DESC` keyword in Apex's lexer in at least some contexts and produces "Unexpected token 'desc'" parse errors. Rename every such local to `describeResult` (or similar) throughout that class.
